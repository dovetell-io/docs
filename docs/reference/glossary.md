---
title: Glossary
description: Shared dovetell terms for public docs and template documentation.
---

## Context debt

The accumulated cost of decisions, assumptions, and operating knowledge that
work depends on but no longer knows how to find.

## Context governance

The practice of keeping project context structured, reviewable, and close to
the work.

## Living context

Repo-owned memory that changes as evidence changes.

## Template

A reusable artifact shape, such as a Markdown file, folder structure, prompt,
checklist, or schema.

## Package candidate

A grouped asset that may become public, sellable, or canonical after review.

## Public output

Reviewed material that is safe to publish as documentation, samples, guides, or
customer-facing template docs.

## Source assets

Reusable Dovetell methodology, scaffolding, templates, schemas, prompts, and
deliverables that may become customer-facing after review.

## Reconciliation

The process of comparing source, local, and public context so drift is visible
before changes are accepted.
