The context governance schema is a Markdown operating surface for AI-assisted
project continuity.

It gives a repo a small set of files that agents and humans can read, update,
and preserve with provenance.

## What it helps with

- orientation at the start of a session
- decision capture
- task continuity
- drift detection
- handoff between people and agents
- public/private context boundaries
- template lineage and resync

## Starter shape

The current starter source lives in the Dovetell assets repository as a package
candidate.

The public docs should explain the concepts and stable usage patterns. The
package source should remain the place where templates, schemas, and release
artifacts are curated.

## What to adopt first

Start with:

- project purpose and boundaries
- session handoff
- active priority pointer
- decisions
- active tasks
- changelog

Add richer governance only when coordination pressure makes it useful.
