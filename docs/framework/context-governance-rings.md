---
title: Context governance rings
description: Context governance rings help classify source material by stability, privacy, and publication readiness.
---

Context governance rings classify material by role and readiness.

They help answer:

- Is this source material?
- Is it local project truth?
- Is it reusable?
- Is it public-safe?
- Is it ready to become a paid template?

## Rings

| Ring | Role | Example |
| --- | --- | --- |
| Core context | Minimum operating system for continuity | tasks, decisions, handoff |
| Supplemental context | Optional reusable modules | risks, personas, processes |
| Project context | Local truth for one repo or initiative | launch notes, repo-specific decisions |
| Working context | Drafts, experiments, unresolved notes | rough prompts, prototypes |
| Public output | Reviewed customer-facing material | public docs, guides, samples |

## How to use the rings

Do not promote material just because it exists.

Promote it when:

- the audience is clear
- the source is private-safe
- the claim is supportable
- the example is reusable
- the destination is known

That destination might be a public guide, a free sample, a paid template, or
private operating documentation.
