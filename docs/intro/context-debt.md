---
title: Context debt
description: Context debt is the accumulated cost of decisions, assumptions, and operating knowledge that are not easy to find or reuse.
---

Context debt is the accumulated cost of knowledge that work depends on but no
longer knows how to find.

It usually shows up as repeated questions:

- Why did we choose this approach?
- Which repo owns this?
- Is this public or private?
- What did the last session change?
- Can this template be reused, or was it specific to one project?
- What should the next person or agent read first?

AI-assisted work makes context debt more visible because sessions can move fast
while memory remains fragmented.

## Common causes

Context debt often comes from:

- decisions made in chat but not recorded
- assumptions embedded in implementation details
- project notes mixed with public docs
- private strategy copied into public surfaces
- templates created from one-off work without review
- undocumented drift between source material and published docs

## Dovetell's approach

dovetell treats context as governed project material.

That means:

- context lives near the work
- source material has provenance
- public documentation is promoted from reviewed source material
- paid templates are packaged from stable, reviewed assets
- drift is reported before anything is overwritten

The point is not to eliminate change. The point is to make change visible and
reviewable.
