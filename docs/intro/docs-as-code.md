title: Docs as code
description: dovetell follows a docs-as-code model for public documentation, template source, and paid documentation packages.

dovetell uses a docs-as-code model.

Documentation is written as Markdown, reviewed in Git, built as a static site,
and promoted deliberately from source material into public or paid surfaces.

## Principles

- Write in Markdown.
- Keep source, review, and publication paths explicit.
- Use Git history for traceability.
- Keep public docs separate from private operating context.
- Promote reusable material only after review.
- Prefer small, task-oriented pages over large essays.

## Style direction

The writing style should follow the spirit of the GitLab Documentation Style
Guide:

- use clear, direct language
- write task-oriented docs
- prefer active voice
- use descriptive headings
- avoid unsupported product claims
- keep examples private-safe

This is guidance, not bureaucracy. The docs should help people move, not admire
the process.

## Publication model

The public docs site is the durable home for dovetell documentation.

The source-of-truth for reusable package material can still live elsewhere,
especially when the material needs privacy review, package curation, or paid
distribution before it becomes public.
