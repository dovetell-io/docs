# tasks.md

Track active commitments only.

Avoid turning this into a backlog landfill. If a task is not likely to matter
soon, leave it out or move it to a separate backlog.

## Active Tasks

- [ ] Replace with a specific active task
  - `id`: task-YYYYMMDD-short-name
  - `owner`: replace-me
  - `status`: open
  - `priority`: now
  - `updated`: YYYY-MM-DD
  - `notes`: Describe the next concrete action, not the whole project history.
  - `done when`:
    - Add one or two clear acceptance checks.

## Completed Tasks

Move completed tasks out of this active file once they no longer help the next
session. Keep a short changelog or archive if the project needs history.
